# Skill: CEO Briefing Generator

> Skill Name: `ceo_briefing.generate`
> Category: Reporting / Business Intelligence
> Type: Scheduled Output
> Trigger: Monday morning (or manually) — also called "Monday Morning Briefing"
> Output: `/Briefings/YYYY-MM-DD_ceo_briefing.md`

---

## Purpose

Generate a concise, executive-readable briefing that gives the "CEO view" of the vault: what was accomplished, what's blocked, what's at risk, and what needs a decision. Transforms the AI from a chatbot into a proactive business partner.

---

## When to Generate

- Every Monday morning (or first session of the week)
- When explicitly requested via a task in `/Inbox`
- Whenever the trust ledger needs a new briefing date entry

---

## Inputs

- Vault folder state (file counts in each folder)
- Recent `/Logs` entries (last 7 days)
- `/Logs/trust_ledger.md` — progress metrics
- `/Done` files — completed work since last briefing
- `/Pending_Approval` — outstanding decisions needed
- `/Review` — unresolved escalations
- `Business_Goals.md` — revenue targets, active projects, subscription audit rules
- (Optional) Financial data files dropped into `/Inbox` by watchers (bank exports, transaction JSON)

---

```markdown

> Generated by: Claude Code

---
```

---

## Steps

1. Read vault folder counts (Inbox, Active, Done, Pending_Approval, Review)
2. Read `/Logs` entries from the past 7 days
3. Read all `/Done` files created since the last briefing
4. Check `/Pending_Approval` for open items and how long they've been waiting
5. Check `/Review` for unresolved escalations
6. Read `/Logs/trust_ledger.md` for Level 5 metrics
7. If financial data files exist in `/Inbox` (e.g., bank exports), parse and include
8. Write the briefing to `/Briefings/YYYY-MM-DD_ceo_briefing.md`
9. Log the generation in `/Logs`
10. Update the trust ledger (briefing date +1)
11. Update `Dashboard.md`

---

## Business Handover (Proactive Suggestions)

When `Business_Goals.md` and financial transaction data are both available, extend the briefing with a **Proactive Suggestions** section:

1. Read `Business_Goals.md` — extract revenue target, active projects, subscription audit rules
2. Run `sentinels/audit_logic.py` against available transaction data to identify:
   - Subscriptions with no activity in 30+ days
   - Cost increases > 20%
   - Monthly software spend over threshold
3. For each flagged item, add a suggestion block:
   ```
   - **[Tool Name]**: [Reason for flag]. Cost: $X/month.
     - [ACTION] Cancel subscription? → Move to /Pending_Approval
   ```
4. Compare `Done` tasks against `Business_Goals.md` active projects — flag any projects behind schedule

**Template**: See `/Templates/monday_briefing.template.md` for the full briefing schema.

---

## Rules

- Never fabricate numbers — only report what the vault data confirms
- Always include the "Decisions Needed" section, even if empty
- Always include a "Proactive Suggestions" section when financial data is available
- Always update the trust ledger after generation
- Briefing file name must include the date for historical trend tracking
- Do not overwrite prior briefings — each date gets its own file
